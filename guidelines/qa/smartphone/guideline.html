    このガイドラインについて

    このガイドラインは既存のアプリやレビュー等を参考に、ユーザーが期待するであろう状態をまとめたものです。 アプリの開発に関わっている方たちが日々の動作確認を実施する中で、こういった点に気付いてほしい、というチェックリストになっています。 ※項目の中にはOSや端末に依存する部分も含まれます。

    インストール

    • 初回インストールが問題なく完了すること
    • 既にインストールされている場合、アップデートが問題なく完了すること

    記録メディア対応(Android)

    • インストール後、記録メディアへアプリが移動できることが望ましい
      ※但し、記録メディアへアプリを移動した場合、一部の機能が制限される(はず)←ホームのウィジェット等

    起動

    • 本体に保存しているアプリが正常に起動できること
    • 記録メディアへ移動しているアプリが正常に起動できること

    ログイン

  • 初回インストール後
    • 登録済みアカウントで利用可能な場合：ログインが可能であること
    • 未登録の場合：新規登録を促す導線があり、登録完了後ログイン可能であること
  • アップデート後
    • 再度ログインを求められないこと(※アップデート前にログインしているので、アップデート後はログインを求めない)
    • アップデート前のデータが引き継がれていること

    ログアウト

    • ログアウトできること
    • ログアウト後、再度ログインできること

    本体メモリ

  • 内部ストレージ
    • アプリの容量が大きすぎないこと
    • アプリの容量が10MBを超える場合はSDカードへ保存可能であること(Androidの場合)
      ※容量が大きい場合、端末によっては他のアプリのインストールやメールの送受信を阻害することがある
  • RAM
    • 使用容量が大きすぎないこと
      ※他のアプリの挙動やバッテリー残量に影響する場合がある

    電波状況

  • 電波状況：良(3～5本)、悪(1～2本)、LTE接続、3G接続
    • 電波状況に関係なく通信できること
      ※気にすること：通信するデータ量
  • 回線契約無し：Wi-Fi接続のみ
    • 通知等で機能が制限される場合はアプリ内にその旨の文言が設けられることが望ましい
  • 圏外
    • 圏外の状態でアプリを起動した場合、通信ができない旨のエラー等が表示されることが望ましい

    バッテリー

  • 残量：100％～25％
    • 正常にアプリが起動すること
    • アプリを起動することによって消費電力が極端にあがらないこと
  • 残量：25％～0％
    • 正常にアプリが使用できること
    • アプリを起動することによって消費電力があがらないこと
    • アップロード、アップデートを行う際にバッテリー残量が少ないお知らせが出ることが望ましい

    本体の発熱

    • アプリ起動中に本体(CPU)が過度に発熱しないこと

    位置情報(iOS)

  • 位置情報サービス
    • 位置情報サービスがOFFに設定されている状態で位置情報を取得する必要がある場合、設定をONに促すダイアログが表示されること
      →ONに設定後、位置情報が取得できること

    位置情報(Android)

  • Googleの位置情報サービス／GPS機能
    • 位置情報サービス、GPS機能がOFFに設定されている状態で位置情報等を取得する必要がある場合、各設定をONにするよう促すダイアログ等が表示されること。
      →ONに設定後、位置情報等が取得できること
    • ダイアログから設定画面へ移行できること。またはダイアログで設定できる場合はONを選択することで設定が可能であること。

    マップ連動

    • 取得した位置情報から正確なマップが表示されること

    通知系

  • Push通知
    • 更新やお知らせがあった場合、PUSH通知が届くこと(通知バーやダイアログが表示されること)
    • PUSH通知の有無を設定内で変更できること
    • PUSH通知をONにしている場合、通知間隔(時間)が設定可能なこと
      ※PUSH通知をONにしている場合、バッテリー消費に影響することがアプリ内(設定等)で説明されていること
  • メール通知
    • 更新やお知らせがあった場合、メール通知が届くこと
    • メール通知の有無を設定内で変更できること
  • アラート通知
    • 通信が失敗した場合、アラート通知が表示されること

    カメラ連動

  • メインカメラ／サブカメラ
    • カメラが起動すること
    • カメラで撮影した画像がアプリ内で確認できること
    • アプリ経由で撮影した画像がカメラロール/ギャラリーに保存されること

    カメラロール／ギャラリー

  • 写真や動画の選択、表示、保存
    • カメラロール/ギャラリーが起動し、写真や動画を選択できること
    • カメラロール/ギャラリー内の写真や動画をアプリ内で表示する場合はすべてのデータが表示され、選択できること
    • アプリ内でダウンロードした画像がカメラロール/ギャラリー内に保存されること
      ※フォルダを新規作成して画像が保存されることが望ましい(Androidの場合)

    電話帳連携

  • 電話帳との同期
    • 電話帳と同期を行う場合は、同意確認を行うこと→同意された場合にのみ電話帳との同期を行うこと
      ※拒否されたら同期は行わない。
      ※初期起動時以外にも設定内で同期(連携)設定が変更できること
  • 電話帳アクセスモニター(SHARP製の一部のAndroid端末)
    • 電話帳アクセスモニター
      ※一部の端末に搭載されている機能を無視していないこと

    電話番号との紐づき

  • アプリを利用する際の電話番号との紐づき
    • アプリを利用する際に電話番号を紐づかせる場合は同意確認を行うこと
    • 電話番号を持たない端末からもアプリを利用できるよう、他アプリやサービスのアカウントと紐づかせて利用できることが望ましい

    ウィルス対策アプリ

    • ウィルス対策アプリがインストールされている端末で、アプリがウィルス検知されないこと

    再インストール

    • 使用端末が変わった場合や、一度アプリを削除してから際インストールした場合、初回登録時のデータが引き続き使用できること

    ディスプレイサイズ

  • 下記のどのサイズにおいても、ユーザーが操作しやすい範囲でメニュー等が配置されていること
    • 3.0～3.9インチ
    • 4.0～4.9インチ
    • 5.0インチ以上

    ディスプレイタイプ(iOS)

  • 下記のどのディスプレイタイプにおいても視認性、操作性に問題がないこと
    ※気にすること：配色、アイコンや画像の見た目や輪郭(にじみやぼやけ等)
    • Retina
    • 非Ratina

    ディスプレイタイプ(Android)

  • 下記のどのディスプレイタイプにおいても視認性、操作性に問題がないこと
    ※気にすること：配色、アイコンや画像の見た目や輪郭(にじみやぼやけ等)
    • IPS
    • ASV
    • CGシリコン
    • IGZO
    • 有機EL

    ディスプレイ解像度

  • 下記のどの解像度においても視認性、操作性に問題がないこと
    ※気にすること：フォントサイズ、ボタンの大きさ　【参考資料はこちら】
    • 480×800／480×854／480×960
    • 540×960
    • 768×1024
    • 720×1280
    • 800×1280
    • 1080×1920

    ディスプレイ輝度

  • 下記のどの設定においても視認性、操作性に問題がないこと
    • 低
    • 中
    • 高

    バックライト消灯

    • アプリ内で動画等を再生する場合、またはゲームをプレイする場合、バックライトは消灯しないこと
      ※視聴や操作に支障をきたすため、消灯しないことが望ましい

    画面回転 (横画面対応)

  • 画面回転：ON
    • アプリが横画面に対応していることが望ましい
    • 横画面に対応している場合、スムーズに回転できること
  • 画面回転：OFF
    • 端末側の設定に倣って画面が回転しないこと

    サウンド

  • サウンドの出力
    • アプリ内で音が鳴る場合は本体の設定に準じること。またはアプリ側で出力の有無が設定可能なこと。
    • 画面ロック中は音が鳴らないこと
    • アプリが音声を再生することを目的としている場合(例：音楽再生アプリ)、画面ロックがかかった場合も音声が継続して再生されること
  • 音声出力の優先
    • 音楽等を再生している状態で音声が出力されるアプリを起動した場合の音声出力の優先度について、
      音楽再生を止めるのか、バックグラウンドで再生を継続するのか等が検討されていること。
      ※バックグラウンドで音楽等の再生を継続させるか否かはアプリの種類によるものと思われるので、
      • アプリ作成時に検討されていることが望ましい

    画面ロック

  • アプリ起動中の画面ロック
    • アプリ起動中に自動で画面ロックがかかること
    • ロックがかかることで不都合が発生するアプリの場合は起動中に画面ロックがかからないこと
      (例：動画再生等)
  • 画面ロックからの復帰
    • (手動・自動ロック関わらず)画面ロックから復帰した場合に、アプリの画面が表示されていること
      ※入力・編集中等の状態がそのまま保持されていることが望ましい

    NFC/FeliCa

    • 対応している端末間でデータの通信ができること
      ※取得したデータ等が表示、利用できること

    赤外線通信(Android)

    • 対応している端末間でデータの通信ができること
      ※取得したデータ等が表示、利用できること

    着信・メール受信

  • アプリ起動中の着信
    • アプリ起動中に着信があった場合、着信が優先されること
    • 通話終了後、入力・編集中等の状態がそのまま保持されていることが望ましい
  • アプリ起動中のメール受信
    • アプリ起動中、メールが受信可能であること
      ※アプリ起動中にバックグラウンドでメールが受信できること

    バックグラウンド

  • バックグラウンドの挙動
    • アップロード、ダウンロードがバックグラウンドで実行できること
      • ※アップロード、ダウンロード中にアプリ内の別画面やホーム画面等へ遷移してもバックグラウンドで実行できること
    • アプリを操作していない状態でバックグラウンドで不要な通信や何らかのサービスが実行されていないこと
      ※バッテリー残量に影響します
      ※但し、音声の再生を目的とするアプリ(例：音楽再生アプリ)の場合は、アプリがバックグラウンドであっても実行されていること

    その他本体側の設定関連

  • 日付と時刻設定
    • 西暦、和暦の設定がアプリへ影響しないこと
    • 時間表示(24時間表示/12時間表示)、日付形式がアプリへ影響しないこと

    本体ボタン(iPhone)

  • 電源ボタン
    • アプリ内のどの場所で押しても画面ロックがかかる
  • ホームボタン
    • アプリ内のどの場所で押してもホームに戻る
  • 音量ボタン
    • 音量を変更できる
  • 音量ボタン
    • 音量を変更できる
  • サイレントスイッチ
    • 音の出力の有無を変更できる

    本体ボタン(Android)

  • 電源ボタン
    • アプリ内のどの場所で押しても画面ロックがかかる
    • 長押しで携帯電話オプションが表示される
  • ホームキー／ボタン
    • アプリ内のどの場所で押してもホームに戻る
    • アプリ内のどの場所で長押ししてもアプリの切り替えができる
  • メニューキー／ボタン
    • アプリのメニューが呼び出せる
  • バックキー／ボタン
    • 一つ前の画面に戻ることができる
  • 音量ボタン
    • 音量を変更できる／スクロールできる
  • カメラキー／ボタン
    • 撮影できる／カメラが起動できる

    テンキー搭載端末

    • テンキーで対応している操作が可能であること

    ディスプレイ操作関連(端末共通)

  • タップ
    • 選択したい場所がタップできること(誤タップがないこと)

    LED点灯

    • 通知等を受け取った際のLED点灯の有無を設定内で変更できること

    対応OSバージョン

  • サポートされているOSが搭載されている端末で問題なくアプリがインストール、起動できること
